📋 项目概览

FR 智能助手是一个面向 FineReport 报表开发场景的 AI 辅助系统，通过多种 AI 技术实现报表需求的智能分析、自动生成和知识管理。
🧠

RAG 知识增强

基于向量检索的 SOP 知识库，自动解析 FR 模板文件（.cpt），构建报表知识图谱，支持智能问答与方案推荐。

Python
📸

OCR 识别

支持 Tesseract 与 Step-1V 双引擎，从图片中提取报表结构和表格数据，自动转换为可编辑的报表模板。

Python
🤖

AI Agent

多版本 Agent 演进（v3→v6→v7），支持需求理解、SQL 生成、报表构建、前后端协作的全自动化流程编排。

Python
🗄️

SQL 引擎

智能 SQL 生成与优化引擎，自动识别数据源结构，根据自然语言需求生成对应的查询语句。

Python
☕

Java Agent 工具

FineReport 进程附加与分析工具集，支持运行时资源解密、类转储、插件热加载等调试与逆向分析能力。

Java
🌐

Web 服务

基于 FastAPI 构建的 RESTful API 服务，提供报表管理、数据源连接、OCR 识别、AI 对话等完整接口。

Python

🔓 开源可行性分析

明确可开源与不可开源的模块划分

✅ 可开源的核心代码

📂 01_knowledge/ — RAG 知识库构建管线
📂 03_rag/ — 提示词模板
📂 04_cpt_engine/ — AI Agent 核心引擎
  ├─ agent.py / agent_v6.py / agent_v7.py — 多版本 Agent
  ├─ pipeline.py — 全流程编排
  ├─ server.py / web_server.py — API 服务
  ├─ cpt_builder.py / v2 / v4 / v5 — CPT 构建器
  ├─ sql_engine.py / sql_skills.py — SQL 智能引擎
  ├─ ocr_skills.py — OCR 识别能力
  ├─ fr_reader.py — FineReport 文件读取器
  ├─ cpt_templates.py — 报表模板管理
  ├─ checker.py — 报表验证器
  └─ web/ — 前端界面
📂 docs/ — 项目文档
📄 *.java — Java Agent 工具（DumpAgent, AttachAgent, ResourceDecryptor）

❌ 不可开源的内容

📦 plugin.jar — FR 官方 AI 插件编译包（11MB）
  涉及 FineReport 商业软件知识产权
📦 dumped.jar — 反编译/转储的 FR 类文件
  包含 FR 内部实现代码
🔐 res/ res_decrypted/ — 加密/解密后的 FR 资源
  包含 FR 商业资源文件
🔑 API Keys — pipeline.py 中的 LLM API 密钥
  (DEEPSEEK_KEY, STEP1V_KEY 等)
📂 02_embedding/model_cache/
  模型缓存文件（可从官方渠道下载）
📂 build/ build3/
  编译产物

📋 开源清理清单

开源前需要从代码中清理以下敏感信息：

1. pipeline.py 中的 STEP1V_KEY 和 DEEPSEEK_KEY — 替换为环境变量引用
2. server.py 中的固定路径 C:\workspace\04_cpt_engine — 替换为相对路径
3. pipeline.py 中的 FR_REPORTLETS 固定路径 — 改为配置化
4. 所有 .jar 编译文件移除（保留源码即可）
5. res/ 加密资源目录移除
6. 添加 .gitignore 排除 model_cache、build、*.jar、res* 等目录
